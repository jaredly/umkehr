examples/block-rich-text: Let's support "inline embeds". This is something that gets treated as a "single character" for the purposes of navigation/selection, but might be rendered in any arbitrary way. An example would be `@mentions` or a `date` embed that shows a given date mm/dd/yyyy. You wouldn't be able to select "within" the embed, but clicking it might open a custom UI for modifying the value, for example a date picker.
One way to implement this would be to have the actual character be some sigil (like `\0` or something), and have a `mark` of type `embed`, with the payload having a `type` and a payload `value`. We would want embeds to be 'pluginable'. And then when rendering, if we encounter a `\0`, we look for the `embed` mark (if it's missing, we render something like "unknown embed") and lookup the renderer (and editor popoverr) for that embed type.
