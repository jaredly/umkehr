# Collaborative whiteboard plan

## Goal

Add a collaborative whiteboard as a new app under `examples/react-crdt/src/apps`. The app should support sticky notes, freehand vector strokes, native emoji stamps, local selection, archived/recoverable elements, zoom/pan with a minimap, and collaboration through the existing React CRDT runtimes.

The first version should target hundreds of elements, not dense design-tool scale.

## Confirmed decisions

- Add an app picker and persist the selected app in `location.hash`.
- Store board elements as `Record<string, WhiteboardElement>`, not an array.
- Put `zOrder: string` on each element and use fractional index values for layering.
- Reuse/export the existing `fractionalIndexBetween` machinery instead of creating a separate z-order array.
- Archive elements instead of deleting them; archived elements should be recoverable.
- Sticky note text commits on blur or Enter, not live per keystroke.
- Moves/resizes use `when: 'preview'` during pointer movement and commit once on pointer up.
- Freehand strokes become committed vector elements only on pointer up.
- Store stroke points as point arrays, simplified before saving.
- Erasing archives whole stroke objects; no partial stroke erasing in v1.
- Final committed content is enough for in-progress strokes; remote in-progress strokes are out of scope.
- Selection should be broadcast through an extended presence API.
- Use native emoji text rendering.
- Include zoom/pan and a minimap.

## Implementation steps

### 1. Expose fractional indexing

In the core CRDT package:

- Export `fractionalIndexBetween` from `src/crdt/index.ts`.
- Keep its type surface simple: callers should pass optional `before` and `after` fractional strings and receive a new string.
- Add or update package smoke tests if needed so the export is covered.

This lets the whiteboard app use the same ordering primitive as CRDT arrays for per-element `zOrder`.

### 2. Add app selection to the shell

In `examples/react-crdt/src/lib/appRegistry.ts` and `examples/react-crdt/src/App.tsx`:

- Register both Todos and Whiteboard.
- Add a compact app picker next to or near the existing mode tabs.
- Store selected app id in `location.hash` along with the selected mode.
- Resolve unknown app ids back to the default app.
- Make sure each selected app passes the matching `AppDefinition`, `CrdtRuntime`, and `HistoryRuntime` into Solo, LocalFirst, Server, PeerJS, and LocalSimulator modes.

This should be generic enough that future example apps can be registered without changing every mode.

### 3. Create the whiteboard model and app adapter

Add:

- `examples/react-crdt/src/apps/whiteboard/model.ts`
- `examples/react-crdt/src/apps/whiteboard/WhiteboardApp.tsx`
- `examples/react-crdt/src/apps/whiteboard/WhiteboardPanel.tsx`

The model should include:

- `WhiteboardState = {background: string; elements: Record<string, WhiteboardElement>}`;
- discriminated element types for `note`, `stroke`, and `emoji`;
- shared fields: `id`, `type`, `x`, `y`, `rotation`, `zOrder`, `createdBy`, `createdAt`, `archived`, `archivedBy?`, `archivedAt?`;
- `typia` schema and validator;
- history and synced contexts using tag key `type`;
- a fresh whiteboard doc id, initial state, and seed timestamp.

Keep ids generated by the app as simple strings safe for record keys.

### 4. Add whiteboard helpers

Add small app-local helpers, likely in `geometry.ts`, `ids.ts`, or `model.ts`:

- `orderedElements(state)` sorts non-archived elements by `zOrder`, then `id`.
- `archivedElements(state)` sorts archived elements by `archivedAt`.
- `nextTopZOrder(elements)` uses `fractionalIndexBetween(top?.zOrder)`.
- `zOrderBetween(before?, after?)` wraps the exported fractional-index helper.
- `screenToBoard(...)` and `boardToScreen(...)` handle pan/zoom coordinate conversion.
- `simplifyStroke(points)` reduces point count before commit.
- `strokePath(points)` converts saved point arrays to SVG path data.

Keep these pure and separately testable where practical.

### 5. Build the whiteboard panel UI

In `WhiteboardPanel.tsx`:

- Render a toolbar with tools: select, note, pen, emoji, archive, recover/trash, undo, redo, zoom controls.
- Render a full board surface with SVG for strokes and selection outlines.
- Render sticky notes as positioned HTML elements so textarea/input behavior is normal.
- Render emoji stamps as native emoji text at board coordinates.
- Keep local UI state for selected tool, selected element id, in-progress stroke, active drag/resize, pan, zoom, and minimap viewport.
- Derive visible elements with `orderedElements`.

Use the existing example styling approach in `examples/react-crdt/src/style.css`, but keep whiteboard-specific class names separate from Todo classes.

### 6. Implement element creation

For each creation command:

- Generate a stable element id.
- Compute board coordinates from the pointer position.
- Assign `zOrder` after the current top visible element.
- Fill `createdBy` from the `actor` prop and `createdAt` from the current wall-clock ISO string.
- Set `archived: false`.

Sticky notes:

- Create a default note at the click point.
- Focus its editor after creation if feasible.
- Commit text on blur or Enter.
- Escape should cancel editing without committing the current draft.

Emoji stamps:

- Start with a small fixed emoji palette.
- Clicking the board places the selected native emoji.

Strokes:

- Collect raw pointer points locally during pointer drag.
- Render the active stroke immediately from local state.
- On pointer up, simplify points, create a stroke element, and commit once.
- Ignore too-short strokes.

### 7. Implement move, resize, archive, and recovery

Move/resize:

- During pointer movement, dispatch preview updates with `when: 'preview'`.
- On pointer up, dispatch one committed update with the final geometry.
- On pointer cancel, Escape, or lost capture, clear the preview.

Archive:

- Set `archived: true`, `archivedBy`, and `archivedAt`.
- Hide archived elements from the main board.
- Do not modify `zOrder`.

Recover:

- Provide a trash/archive panel or popover listing archived elements.
- Clear `archived`, `archivedBy`, and `archivedAt`.
- The recovered element should return to its previous layer because `zOrder` was preserved.

Whole-stroke erasing:

- In erase mode, hit-test strokes as whole objects and archive the selected stroke.
- Partial stroke erasing is out of scope.

### 8. Implement layering controls

Layer operations update only `elements[id].zOrder`.

Support at least:

- bring to front;
- send to back;
- bring forward one step;
- send backward one step.

Compute neighboring visible elements from `orderedElements(state)` and use fractional indexes to place the selected element between neighbors. Resolve ties in rendering by sorting by `id`.

### 9. Add zoom, pan, and minimap

Zoom/pan:

- Maintain `zoom` and `pan` as local state.
- Support wheel/trackpad zoom where reasonable.
- Support drag-to-pan with a dedicated hand mode or modifier.
- Keep pointer hit testing routed through `screenToBoard`.

Minimap:

- Render a small overview of visible elements.
- Show the current viewport rectangle.
- Allow clicking or dragging in the minimap to recenter the board if straightforward.

The viewport is local UI state and should not be stored in the CRDT document.

### 10. Extend presence for selection broadcast

Add an example-level ephemeral presence path for selection, separate from document updates.

Target behavior:

- When a user selects an element, other users can see a remote selection indicator for that element.
- Selection should be keyed by actor/session, not written into the CRDT document.
- Clearing local selection should clear the remote selection status.

Likely implementation:

- Extend server presence protocol with a generic presence/status payload, or a specific whiteboard selection message.
- Add matching client-side handling in `useServerSync`.
- Store remote selections in the existing `StatusStore`, targeting `elements[id]`.

Local simulator and PeerJS parity can follow later if the server path is enough for the first pass.

### 11. Performance pass for hundreds of elements

Before calling v1 done:

- Memoize ordered element derivation where useful.
- Avoid measuring every element on every pointer move.
- Keep pointer-move work small and rely on preview dispatch for document-shaped rendering.
- Simplify strokes before saving.
- Ensure SVG path rendering remains responsive with hundreds of elements.

Virtualization is probably unnecessary for hundreds of elements, but avoid designs that assume only a handful.

### 12. Verification

Run the relevant checks:

```sh
pnpm --dir examples/react-crdt build
```

If package-level tests are available and affected by exporting `fractionalIndexBetween`, run the relevant test command for the root package too.

Manual smoke test:

- Switch between Todo and Whiteboard with the app picker and verify `location.hash` updates.
- In solo mode, create notes, strokes, and emoji stamps.
- Commit note text with blur and Enter.
- Move and resize elements with preview, then undo/redo the committed operation.
- Archive and recover each element type.
- Recover an archived element and confirm it returns to its prior layer.
- Bring elements forward/backward and verify ordering is stable after reload.
- Draw strokes and confirm saved point counts are simplified.
- Erase a stroke and confirm it archives the whole stroke.
- Zoom and pan; verify hit testing remains aligned.
- Use the minimap to inspect/recenter the viewport if implemented.
- In local simulator mode, create/edit/archive/recover from both panels with sync enabled and paused.
- In server mode, verify remote selection presence appears and clears.
- Reload server/local-first mode and confirm the board persists.
- Create hundreds of mixed elements and confirm interaction remains usable.

## Out of scope

- Partial stroke erasing.
- Remote in-progress stroke streaming.
- Character-level collaborative text inside sticky notes.
- Rich text formatting in notes.
- Cross-platform emoji image normalization.
- Durable viewport/pan/zoom sync.
- Full design-tool scale optimization.
