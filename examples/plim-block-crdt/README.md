# Plim Block CRDT Example

This example uses Plim as the editable React view for `umkehr/block-crdt`.

The block CRDT state is authoritative. Plim handles DOM editing, commands,
keyboard behavior, slash commands, marks, and React integration, but document
changes are translated into CRDT `Op[]`, applied to the CRDT state, and then
rematerialized back into a canonical Plim `EditorState`.

Plim block ids are CRDT Lamport strings after each adapter cycle. Temporary ids
that Plim creates during split or insert transactions are intentionally replaced
with the ids generated by the CRDT.

## Running

From the repository root:

```sh
npm install
cd examples/plim-block-crdt
npm run dev
```

Useful checks:

```sh
cd examples/plim-block-crdt
npm run test
npm run build
```

The root package also includes this example in `npm run typecheck:examples`.

## What It Demonstrates

- Two side-by-side Plim editors backed by independent block CRDT replicas.
- Online sync for local edits, plus offline queuing and flushing when replicas
  reconnect.
- Text insertion and replacement with grapheme-correct offset conversion.
- Block split, join, insert, remove, and move translation.
- Block metadata updates for Plim block type and attrs.
- Bold, italic, code, and link mark registration, including toolbar and
  keyboard shortcut paths.
- Multi-block mark toggles emitted by Plim as per-block mark operations.
- Slash command integration through Plim's slash command extension.
- Retained selections that survive CRDT rematerialization and remote edits.
- Demo undo/redo implemented by planning inverse CRDT operations and syncing
  those inverse operations to the peer replica.
- Debug panes showing the materialized Plim JSON and CRDT text for each
  replica.

## Adapter Model

The central loop is:

1. Render the current CRDT state as a Plim document.
2. Let Plim produce transactions from editing commands.
3. Translate supported Plim transaction ops into block CRDT operations.
4. Apply those operations to the local CRDT state.
5. Rematerialize the CRDT state back into Plim document and selection state.
6. Send the CRDT operations to the peer replica, or queue them while offline.

Supported Plim transaction kinds include:

- `replaceText`
- `splitBlock`
- `joinBackward`
- `insertBlock`
- `removeBlock`
- `moveBlock`
- `setBlockType`
- `setBlockAttrs`
- `toggleMark`
- `setSelection`

`setSelection` is local view state and does not emit CRDT operations. It is
converted into a retained CRDT selection so future rematerialization can resolve
the selection through remote inserts, deletes, splits, joins, and moves.

Unsupported transaction ops are collected and rolled back by rematerializing
from CRDT state.

## Important Files

- `src/App.tsx` wires Plim transactions into the adapter and renders both demo
  replicas.
- `src/plimBlockCrdtAdapter.ts` converts CRDT state to Plim documents,
  translates Plim transactions to CRDT operations, and resolves retained
  selections back into Plim selections.
- `src/plimDemoRuntime.ts` manages the two-replica demo, online/offline queues,
  sync, undo, and redo.
- `src/fixtures.ts` builds the initial CRDT document used by the app and tests.
- `src/plimBlockCrdtAdapter.test.ts` covers conversion and adapter behavior.
- `src/App.test.tsx` covers the end-to-end demo behavior in React.
- `ARCHITECTURE.md` has the detailed adapter design notes.

## Known Limits

- This is an integration spike for Plim and `umkehr/block-crdt`, not a packaged
  collaboration provider.
- Remote presence UI is not implemented. The example retains each editor's own
  selection, but it does not render peer cursors.
- Offline sync is an in-memory demo queue, not durable networking or storage.
- Demo undo/redo covers operations that the block CRDT can invert. Unsupported
  inverse plans are reported in the log and skipped.
- `removeBlock` uses block CRDT block-only deletion semantics, so visible
  children are spliced upward instead of deleted with the parent.
- Atomic block types are materialized without text content:
  `divider`, `image`, `embed`, `raw_html`, and `table`.
