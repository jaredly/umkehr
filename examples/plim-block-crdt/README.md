# Plim Block CRDT Example

This example uses Plim as the editor view for `umkehr/block-crdt`.

The CRDT state is authoritative. Plim transactions are translated into CRDT `Op[]`, applied to the CRDT state, then rematerialized back into a Plim document. Plim block ids are CRDT Lamport strings after every adapter cycle; temporary ids generated by Plim during split/insert are intentionally discarded.

Current scope:

- text replacement
- block split/join
- block insert/remove/move
- block metadata updates
- single-block mark toggles
- grapheme-correct offset conversion
- local retained selection after rematerialization and scripted remote ops

Known limits:

- Undo/redo is intentionally skipped for this spike.
- Collaborative remote presence is not implemented.
- Unsupported Plim transaction ops are rolled back by rematerializing from CRDT state.
- `removeBlock` uses block CRDT block-only deletion semantics, so visible children are spliced upward instead of deleted with the parent.
